全局架构决策

引擎策略

兼容 Legado JSON 格式，自建执行。复用 5400+ 书源选择器。

响应式方案

Tailwind CSS + 移动优先断点。<768 单栏 / 768-1024 可选侧栏 / >1024 双栏

手势处理

@use-gesture/react — 翻页滑动、漫画缩放，触摸+鼠标统一

离线存储

Service Worker + IndexedDB (idb-keyval) — 客户端缓存章节和图片

PWA

vite-plugin-pwa — standalone 模式，横竖屏均支持

漫画性能

react-virtuoso 虚拟滚动 + 预加载 — 条漫长列表不卡

导出

后端 ebooklib (epub) + 纯文本拼接 (txt)

部署

Docker 单容器，FastAPI 静态 serve 前端产物

阅读器设计

小说阅读

📜 滚动模式

  • 连续滚动，章节无缝衔接
  • 滚动到底自动加载下一章
  • 进度条百分比
  • 适合快速浏览

📖 翻页模式

  • 左右滑动 / 点击翻页
  • 内容动态分页（按屏幕尺寸计算）
  • 翻页动画（可关闭）
  • 适合沉浸阅读

漫画阅读

📱 条漫模式（Webtoon）

  • 图片垂直拼接连续滚动
  • 虚拟滚动（只渲染可视区域）
  • 预加载前后 3-5 张
  • 双击放大

📚 页漫模式（Manga）

  • 左右翻页（可设阅读方向）
  • 桌面端可选双页并排
  • 捏合/双击缩放
  • 预加载前后 2 页

分阶段实施

Phase 1 — 骨架 + 引擎 + 基础阅读

  • 后端：FastAPI + SQLite + 规则引擎
  • 引擎：CSS/XPath/JSONPath/Regex/简单 JS
  • API：书源管理 + 搜索 + 内容获取
  • 前端：响应式骨架 + PWA 基础
  • 阅读：小说滚动模式（先跑通全流程）
  • 缓存：IndexedDB 已读章节
  • 设置：字号/行距/主题
  • 部署：Dockerfile

Phase 2 — 完整阅读体验

  • 翻页模式（手势 + 动画 + 分页算法）
  • 漫画阅读器（条漫 + 页漫）
  • 整本下载 + 进度显示
  • 导出 epub / txt
  • 书架管理（收藏/分类/更新检测）
  • 阅读进度多设备同步

Phase 3 — TTS + 有声书 + RSS

  • TTS 多后端（Edge/sherpa/API）
  • WebSocket 流式音频 + 高亮
  • 有声书源解析 + 播放器
  • RSS 管理 + 文章阅读
  • 图片防盗链处理增强

Phase 4 — 打磨

  • JS bridge 完整实现（crypto/put/get）
  • WebView 规则（可选 headless）
  • 阅读统计
  • 多用户支持
  • 自动更新通知
